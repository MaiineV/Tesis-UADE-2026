2. Contradicciones (doc vs código)

Ninguna es un bug — el código es la versión correcta. El costo es que cualquier dev (o agent) que lea TECHNICAL.md va a escribir código contra APIs que ya no existen.

CRÍTICA — §14.7 RulesetSO: spec monolítico de 20+ campos vs esqueleto de 4 submódulos

P: El doc lista MaxEnergy, curvas de scaling (EnemyHPMultiplier, GoldReward…), StartingFloor, ForbiddenActionIds, etc. A: Assets/Scripts/Rollgeon/Balance/RulesetSO.cs solo tiene EnergyConfig, TurnOrderConfig, WeaknessConfig, ComboCountersConfig; RollConfig (T100b) y ScalingConfig (T99) marcados "pendiente" en comentarios. S: Reescribir §14.7 al modelo de submódulos y crear ticket para ScalingConfig — sin él no hay dificultad progresiva entre pisos.

ALTA — §1.1.1 ServiceBootstrapSO: campos tipados vs listas genéricas + reflection

P: Doc describe campos explícitos (EntityCatalog, ComboCatalog…). A: El código usa Catalogs: List<BaseCatalogSO> + SettingsAssets + RegisterByRuntimeType() por reflection, y RegisterRunScoped() post-run. S: Reescribir §1.1.1; documentar también la gotcha de Odin (field initializers no corren en ExtraServices deserializados).

ALTA — §13.5 FloorLayoutSO: listas por tipo vs modelo de Slots

P: Doc describe CombatRooms/ShopRooms/BossRooms + RoomCountMin/Max. A: El código (Feature#0009 + FloorEditor) usa Slots: List<RoomTypeSlot> con RoomCountSpec (Fixed/Random) y campo NextFloor para progresión. S: Reescribir §13.5 contra Assets/Scripts/Rollgeon/Dungeon/FloorLayoutSO.cs.

MEDIA — §1.3 CombatTurnFSM: 12 estados flat vs FSM jerárquica

P: Doc documenta ~12 estados granulares (Roll, Reroll, ResolveCombo…). A: El código tiene 4 estados top-level (Enter/PlayerTurn/EnemyTurn/Exit) con sub-FSM dentro de PlayerTurnState. S: Actualizar §1.3 con el diagrama jerárquico real.

MEDIA — §14 Meta-progresión: nombres distintos a los spec'd

P: Doc: BaseUnlockSO, UnlockStateSO, RunRecord. A: Código (Feature#0010): UnlockDefinitionSO, MetaProgressionState+RunUnlockState, servicios MetaProgressionService/UnlockProgressService; RunRecord no existe. S: Alinear §14 al naming real y decidir si RunRecord (historial de runs) se necesita o se descarta.
