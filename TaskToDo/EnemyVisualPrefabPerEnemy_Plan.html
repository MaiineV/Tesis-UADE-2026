Per-Entity Visual Prefab

Cada EnemyDataSO y cada ClassHeroSO referencia su propio prefab visual. Se elimina la heurística central BaseHP >= 80 y los 3 slots (_heroPrefab / _enemyPrefab / _bossPrefab) del bootstrap.

1. Objetivo

  • Cada enemigo decide qué GameObject se instancia como su pawn visual.
  • Cada hero (clase) también decide su propio prefab — mismo patrón que enemies.
  • Remover el branching central "si HP >= 80 entonces boss prefab".
  • Remover los 3 slots del bootstrap: _heroPrefab, _enemyPrefab, _bossPrefab (legacy del FP).
  • Conservar el contrato del EntityVisualService y de EntityPawn — la capa de lógica no cambia.

Fuera de scope: Animaciones, materials, VFX per-entity. Pool de pawns.

2. Estado actual

2.1. Punto central a remover

Assets/Scripts/Rollgeon/Entities/Visuals/EntityVisualService.cs:81-89

public EntityPawn SpawnEnemy(Guid guid, EnemyDataSO data, GridCoord coord)
{
    var isBoss = data != null && data.BaseHP >= 80; // heurística simple para FP
    var prefab = isBoss
        ? (_bossPrefab ?? _enemyPrefab ?? CreatePrimitiveFallback(PrimitiveType.Cube, Color.magenta))
        : (_enemyPrefab ?? CreatePrimitiveFallback(PrimitiveType.Capsule, Color.red));
    var kind = isBoss ? EntityPawn.PawnKind.Boss : EntityPawn.PawnKind.Enemy;
    return SpawnInternal(guid, prefab, coord, kind);
}

El comentario "heurística simple para FP" marca esto como deuda técnica desde el First Playable. Hoy con dos enemy types nuevos (ED_Healer, ED_RangedEnemy) ya no escala.

2.3. EnemyDataSO / ClassHeroSO — falta el campo visual

  • Assets/Scripts/Rollgeon/Entities/EnemyDataSO.cs — tiene Identity, Weakness, Stats, Rewards, Behaviors, AIRoot. Sin VisualPrefab.
  • Assets/Scripts/Rollgeon/Heroes/ClassHeroSO.cs — tiene Identity, Sheet, PhaseBehaviors, DiceBagPool, etc. Recibe el SO en SpawnHero(guid, hero, coord) pero el service lo ignora y usa _heroPrefab del bootstrap.

3. Diseño propuesto

Antes deprecado

El bootstrap decide qué se instancia.

EnemyDataSO  ──┐
               ├──> SpawnEnemy
GridCoord    ──┘        │
                        ▼
               if BaseHP >= 80
                ? _bossPrefab
                : _enemyPrefab

ClassHeroSO  ──┐
               ├──> SpawnHero
GridCoord    ──┘        │
                        ▼
               _heroPrefab  (hero SO ignorado)

Después target

Cada SO trae su prefab.

EnemyDataSO.VisualPrefab ─┐
                          ├──> SpawnEnemy
GridCoord                ─┘        │
                                   ▼
                       Instantiate(data.VisualPrefab)

ClassHeroSO.VisualPrefab ─┐
                          ├──> SpawnHero
GridCoord                ─┘        │
                                   ▼
                       Instantiate(hero.VisualPrefab)

3.3. EntityVisualService simplificado

public EntityPawn SpawnEnemy(Guid guid, EnemyDataSO data, GridCoord coord)
{
    if (data == null) throw new ArgumentNullException(nameof(data));
    if (data.VisualPrefab == null)
    {
        Debug.LogError($"[EntityVisualService] EnemyDataSO '{data.name}' " +
                       "no tiene VisualPrefab asignado.");
        return null;
    }
    var kind = data.IsBoss ? EntityPawn.PawnKind.Boss : EntityPawn.PawnKind.Enemy;
    return SpawnInternal(guid, data.VisualPrefab, coord, kind);
}

public EntityPawn SpawnHero(Guid guid, ClassHeroSO hero, GridCoord coord)
{
    if (hero == null) throw new ArgumentNullException(nameof(hero));
    if (hero.VisualPrefab == null)
    {
        Debug.LogError($"[EntityVisualService] ClassHeroSO '{hero.name}' " +
                       "no tiene VisualPrefab asignado.");
        return null;
    }
    return SpawnInternal(guid, hero.VisualPrefab, coord, EntityPawn.PawnKind.Hero);
}
