WallOccluder × Room Editor — Plan

Auto-asignar WallOccluder + Direction a los tiles pintados de tipo Wall en el Room Editor. Híbrido: auto-paint + bake button Branch: Fix#NNNN_WallOccluderAutoBake

1 · Problema

El sistema de wall occlusion (§17.E.8) requiere que cada pared en una sala lleve un componente WallOccluder con su Direction (N/NE/E/SE/S/SW/W/NW) seteada. Cuando las salas se construían a mano, esto era manejable. Con el Room Editor pintando decenas de wall tiles por sala, asignar componente + dirección uno por uno es tedioso y propenso a olvidos.

El CameraService.RefreshWallOcclusion() (Assets/Scripts/Rollgeon/Camera/CameraService.cs:437) sólo procesa los WallOccluder que encuentra; si falta el componente, la pared no se oculta. Direction mal seteada → la pared correcta no se oculta o se oculta la equivocada.

3 · Decisión & convenciones

Wall prefab

No es required que el prefab traiga WallOccluder pre-bakeado. La tool agrega el componente si falta. Hace al sistema robusto frente a prefabs nuevos, importados o legacy.

Orientación de pared

Confirmado: un solo prefab rotado con R al pintar. La fuente de verdad para Direction es la posición de la celda relativa al centro de la sala, no la rotación del paint (la rotación puede no coincidir 1:1 con la dirección semántica).

UX

Híbrido: asignación automática al pintar (zero clicks) + botón Bake Wall Occluders que re-evalúa todo después de mover/borrar tiles. Mismo patrón que Bake NavGraph.

4 · Algoritmo de inferencia de Direction

Dado una celda (x, z) y un room, calculamos: 

// 1. Centro semántico de la sala = centro del AABB de los tiles Floor.
//    (LocalBounds de RoomLayout puede estar stale entre paints; los
//     Floor markers son verdad fresca.)
center = mean(floor_tile_cells)

// 2. Vector desde center hacia la celda de la pared.
dx = cell.x - center.x
dz = cell.z - center.z

// 3. Ángulo desde +Z (norte), CW = positivo.
//    atan2(dx, dz) en radianes → grados [0, 360).
angle = (Mathf.Atan2(dx, dz) * Rad2Deg + 360f) % 360f

// 4. Cuantizar a 8 octantes (cada uno = 45°), centrados en cada
//    direccional cardinal/diagonal.
octant = Mathf.RoundToInt(angle / 45f) % 8
direction = (WallDirection)octant  // 0=N, 1=NE, 2=E, ... 7=NW

Casos borde

  • Sala sin floors aún (paintamos walls primero): fallback a RoomLayout.LocalBounds centro. Si tampoco hay, Direction.N con un Debug.LogWarning sugiriendo ejecutar Bake Wall Occluders al final.
  • Pillar interior (pared no en el borde de la sala): el algoritmo de octantes igual entrega la dirección "más alejada del centro". Razonable como default; el usuario puede override manual desde el inspector del WallOccluder — no lo pisamos al re-bakear si tiene un flag ManualOverride (ver §6).
  • Celda exactamente en el centro (angle == 0 y dx == dz == 0): N por convención.

5.2 · Hook al pintar

En RoomEditorWindow.PlaceAt (línea 855), después del bloque que maneja TileType.Door: 

if (entry.Type == TileType.Wall)
{
    WallOccluderOps.EnsureOccluder(instance, _target, cell);
}

El helper hace Undo.AddComponent + Undo.RecordObject + EditorUtility.SetDirty, así que el paint sigue siendo reversible con un solo Ctrl+Z.

5.3 · Botón "Bake Wall Occluders"

En RoomEditorWindow.cs, debajo de BakeNavGraph (línea 267), agregamos: 

[TabGroup(Tabs, TabRoom), BoxGroup(GRoom, false), PropertyOrder(10)]
[Button("Bake Wall Occluders", ButtonSizes.Medium), EnableIf("@_target != null")]
private void BakeWallOccluders()
{
    if (_target == null) return;
    Undo.RecordObject(_target, "Bake Wall Occluders");

    int added = 0, updated = 0, skipped = 0;
    var markers = _target.GetComponentsInChildren<TileMarker>(true);
    foreach (var m in markers)
    {
        if (m.Type != TileType.Wall) continue;
        var cell = new Vector3Int(m.Coord.X, m.Layer, m.Coord.Y);
        var (a, u) = WallOccluderOps.EnsureOccluder(m.gameObject, _target, cell);
        if (a) added++;
        else if (u) updated++;
        else skipped++;
    }
    Debug.Log($"[WallOccluder] Baked: {added} added, {updated} updated, {skipped} ok.");
}

5.4 · Override manual (opcional, recomendado)

Sumar [SerializeField] private bool _manualOverride; en WallOccluder + chequearlo en EnsureOccluder para no pisar la Direction de paredes que el usuario ajustó a mano (típicamente pillars internos).

9 · Riesgos & mitigaciones

  • Salas no rectangulares / L-shaped: el centro promediado puede caer fuera del polígono real. Mitigación: el bake es idempotente — si un wall queda con Direction rara, el usuario la corrige a mano + activa ManualOverride.
  • Multi-layer (Y > 0): el algoritmo opera en plano XZ (ignoramos cell.y). Una pared en layer 2 igual recibe Direction calculada desde la posición XZ, que es lo correcto para ocluder en cámara isométrica.
  • Performance: bake recorre todos los markers; en una sala de 200 tiles es < 1ms. No hace falta caching.
  • Conflicto con Undo: EnsureOccluder usa Undo.AddComponent + Undo.RecordObject → un único Ctrl+Z deshace el paint completo (instancia + occluder).

10 · Preguntas abiertas

Q1 — ¿Aplica también a TileType.Door?

Las puertas son visualmente "paredes con hueco" pero no se ocluyen (el jugador necesita verlas para navegar). Recomendación: NO. El hook sólo dispara en TileType.Wall.

Q2 — ¿ManualOverride flag o tag separado?

Alternativa: en vez de flag, marcamos el GameObject con un Editor-only tag o un componente vacío WallOccluderLocked. Más "explícito" en el inspector pero más fricción. Recomendación: flag simple en el WallOccluder.

Q3 — Indicador visual en SceneView durante el paint

¿Querés que el ghost del Room Editor pinte una flechita o letra (N, NE, ...) sobre la celda cuando el tile pintable sea TileType.Wall? Es lindo feedback pero requiere extender RoomEditorGhost. Sugerido como follow-up, no parte del MVP.

Q4 — ¿El centro de la sala debe ser cell o world?

Recomendado en grilla (cell coords) para evitar problemas si el GridOrigin está desplazado. El algoritmo trabaja con ints/floats sin tocar Transforms.
